\doxysubsection{Detailed Description}
The {\ttfamily \mbox{\hyperlink{classcom_1_1pekinsoft_1_1loadmaster_1_1controller_1_1BrokerCtl}{Broker\+Ctl}}} object provides all functionality related to accessing the {\ttfamily brokers.\+tbl} database file. 

This object provides the ability to access all of the records in the table, navigate between those records, as well as to search those records.

When creating an object of type {\ttfamily \mbox{\hyperlink{classcom_1_1pekinsoft_1_1loadmaster_1_1controller_1_1BrokerCtl}{Broker\+Ctl}}}, the developer {\bfseries{must}} enclose the call to the constructor within a {\ttfamily try...catch()} block, as the constructor could throw a {\ttfamily Data\+Store\+Exception} while the object is being created. Calls to any of the methods that access the database, likewise, need to be enclosed in {\ttfamily try...catch()} blocks, as all database access methods could throw the {\ttfamily Data\+Store\+Exception} if there are any errors accessing the data file.


\begin{DoxyDescription}
\item[Developer\textquotesingle{}s Note]Pekin\+S\+O\+F\+T™ Sytems {\itshape does not} handle any {\ttfamily Data\+Store\+Exception} trapping within this object. The reason for this is that we believed that it should be up to the individual class developer to determine how those errors should be handled. Pekin\+S\+O\+F\+T™ could not possibly determine the various different ways that this class could possibly be used, and so also couldn\textquotesingle{}t make a quality decision on how to handle errors if they were to arise. For these reasons, we have left the error trapping and handling up to the developers who choose to use our controller objects in their applications and projects.
\end{DoxyDescription}
