\doxysubsection{Detailed Description}
{\ttfamily \mbox{\hyperlink{classcom_1_1pekinsoft_1_1loadmaster_1_1api_1_1AbstractJournal}{Abstract\+Journal}}} is a generic class for use in maintaining the data files and objects for Load Master™. 

By using reflection, we will only have this one class to maintain in the future for all data file access.

Though most people organize their data files in such a way that makes sense to a person reading it in a text editor, Load Master™ will not need to do so. As an example, a human would format the data file for a contact such as the following, using an asterisk as a field delimiter\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{keywordtype}{id}*lastName*firstName*street*apartment*city*state*ZipCode*phoneNumber*email}
\end{DoxyCode}


Since the computer is not human, and does not look at things the same way a human does, we don\textquotesingle{}t need to worry about the format, other than that the system reads in the data in the same order as it was written out. In order to accomplish this, {\ttfamily \mbox{\hyperlink{classcom_1_1pekinsoft_1_1loadmaster_1_1api_1_1AbstractJournal}{Abstract\+Journal}}} sorts the methods by name so that the setter methods will be called in the same order as the getter methods. Therefore, using the example above, the data file will be formatted as such\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{aprtment*city*email*\mbox{\hyperlink{classcom_1_1pekinsoft_1_1loadmaster_1_1api_1_1AbstractJournal_ad099c218488c03f224be3e2efbeb1a78}{first}}*\textcolor{keywordtype}{id}*\mbox{\hyperlink{classcom_1_1pekinsoft_1_1loadmaster_1_1api_1_1AbstractJournal_ae7db2cffea5f9a9702e2cee29c08bfaf}{last}}*phone*state*street*ZipCode}
\end{DoxyCode}


Though this format is not easy for a human to read, it will work for the system none-\/the-\/less.


\begin{DoxyDescription}
\item[Developer\textquotesingle{}s Note]By using this single class for accessing all of the data files for Load Master™, we must require each model class to pass the full path and file name to the data file for that model into the constructor for {\ttfamily \mbox{\hyperlink{classcom_1_1pekinsoft_1_1loadmaster_1_1api_1_1AbstractJournal}{Abstract\+Journal}}}. In this way, the generic, reflective class will be able to open the file for input/output as needed. 
\begin{DoxyDescription}
\item[Developer\textquotesingle{}s Note]Every model class for which you want to use the {\ttfamily \mbox{\hyperlink{classcom_1_1pekinsoft_1_1loadmaster_1_1api_1_1AbstractJournal}{Abstract\+Journal}}} for data file access must implement the \begin{DoxySeeAlso}{See also}
{\ttfamily \mbox{\hyperlink{interfacecom_1_1pekinsoft_1_1loadmaster_1_1api_1_1JournalInterface}{Journal\+Interface}}} interface. This allows {\ttfamily \mbox{\hyperlink{classcom_1_1pekinsoft_1_1loadmaster_1_1api_1_1AbstractJournal}{Abstract\+Journal}}} to transfer the data from the respective data file to the \mbox{\hyperlink{namespacecom_1_1pekinsoft_1_1loadmaster_1_1model}{model}}, and from the \mbox{\hyperlink{namespacecom_1_1pekinsoft_1_1loadmaster_1_1model}{model}} to the respective data file. By managing data transfer between the \mbox{\hyperlink{namespacecom_1_1pekinsoft_1_1loadmaster_1_1model}{model}} and the data file in this manner, the output of the data file will be organized in the manner directed by the developer.
\end{DoxySeeAlso}

\end{DoxyDescription}
