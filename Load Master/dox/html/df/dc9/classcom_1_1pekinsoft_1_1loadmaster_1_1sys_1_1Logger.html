int  level 
)

Creates a Logger object that places the log file in the path specified by the calling application, logs all messages at the provided logging level or higher and is appended to on future runs of the application.

The valid levels are as follows:

  • DEBUG: Lowest level of logging. All messages sent to the Logger will be written to the log file.
  • CONFIG: Stores configuration messages to the log file.
  • INFO: Stores informational messages to the log file.
  • WARN: Stores messages to the log file that warned users of inappropriate use, bad data, invalid settings, etc.
  • ERROR: Stores messages from recoverable errors to the log file.
  • CRITICAL: Stores messages from non-recoverable errors to the log file to keep track of errors and/or bugs that cause the application to fail and exit abnormally.
Parameters
pathThe path to the folder where the log file is to be placed.
levelThe minimum level at which to write messages to the log.

void com.pekinsoft.loadmaster.sys.Logger.critical ( LogRecord  record)

Logs a system critical error to the log file just before the application exits.This should only be used for unrecoverable errors in the program.

All other errors should be logged through the error method.

The extraData parameter should contain information pertinent to the user within the context of your application. The error logging method places system and Java information into the message by default. All of this extraneous information should aid the software designers, programmers and engineers to be able to track down the error to a specific cause, thereby allowing them to correct the application easier and quicker.

When setting up your LogRecord for a call to this method, follow these guidelines:

LogRecord.setSourceClassName()
This is used for exactly what the method says, the source class name where the Exception was thrown.
LogRecord.setSourceMethodName()
This is, likewise, used for what the method says, the source method name where the Exception was thrown
LogRecord.setThrown()
Again, use this for exactly what the method says, store the actual Exception that was thrown
LogRecord.setMessage()
Use this method to store the package in which the Exception was thrown, e.g., "com.myapp.mypkg"
LogRecord.setResourceBundle(String)
Use this method to pass the edition of the application that is being tested or run: i.e., Basic Edition, Corporate Edition, Enterprise Edition
LogRecord.setSequenceNumber()
Use this method to provide the build number that is being run at the time of the Exception being thrown
LogRecord.setLoggerName()
Use this method to provide the version that is being run at the time of the Exception, i.e. "1.2.19", "0.2.5", etc.
Logger.setParameters(Object[])
Use this method to provide a list of the installed modules when the exception was thrown
Parameters
recordA LogRecord containing all pertinent information as described above.
